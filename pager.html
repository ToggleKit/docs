Pager

A pager divides a single article into multiple pages. This is particularly useful for very long articles that might take a long time to load in full. By breaking the content into smaller sections, a pager can help manage loading times more efficiently and improve the readerâ€™s experience. Additionally, it allows readers to navigate through the content more easily, focusing on specific sections without being overwhelmed by the entire article at once. This method also reduces the strain on server resources, ensuring smoother performance for websites with heavy traffic.

Structure

<div data-tg="pager">
  <div data-page>
  <!--Content-->
  </div>
  <div data-page>
  <!--Content-->
  </div>
</div>

Selectors

[data-tg="pager"] = main container
[data-page] = pages
[data-page][data-pageon] = visible page
[data-tg="numberDiv"] = page navigator number div
[data-tg="pageNumber"] = page navigator button

Note: always first page is visible by default
